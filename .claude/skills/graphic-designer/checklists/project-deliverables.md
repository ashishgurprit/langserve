# Project Deliverables Checklist

## Universal Requirements (All Projects)

- [ ] **Source Files**: Working files in native format (AI, PSD, INDD, Figma)
- [ ] **Export Formats**: Client-requested formats (PDF, PNG, JPG, SVG)
- [ ] **Documentation**: Usage guidelines, specifications document
- [ ] **Version Control**: Clearly named files (no "final_FINAL_v3")
- [ ] **Backup**: Files stored in at least 2 locations
- [ ] **License/Rights**: Usage rights documented (client owns, designer retains, etc.)

---

## Brand Identity System

### Core Files
- [ ] **Logo System**:
  - [ ] Primary logo (full lockup)
  - [ ] Secondary logo (simplified version)
  - [ ] Icon/symbol (standalone mark)
  - [ ] Wordmark (text-only version)
  - [ ] Monochrome versions (black, white, single color)
- [ ] **Source Files**: AI/SVG (vector, editable)
- [ ] **Export Formats**:
  - [ ] PNG (transparent background): 1024×1024, 512×512, 256×256, 128×128, 64×64
  - [ ] JPG (white background): 1024×1024
  - [ ] SVG (web-ready, optimized)
  - [ ] EPS (print-ready)
  - [ ] PDF (vector, high-res)

### Brand Guidelines Document
- [ ] Logo usage (dos and don'ts)
- [ ] Clear space rules (minimum padding)
- [ ] Minimum size specifications
- [ ] Misuse examples (what NOT to do)
- [ ] Color palette (see below)
- [ ] Typography system (see below)
- [ ] Image style guidelines
- [ ] Application examples

### Color System
- [ ] **Primary Palette**: 3-5 brand colors with:
  - [ ] Hex codes (web: #FF5733)
  - [ ] RGB values (screen: R:255 G:87 B:51)
  - [ ] CMYK values (print: C:0 M:66 Y:80 K:0)
  - [ ] Pantone/spot colors (if applicable: PMS 2035)
- [ ] **Extended Palette**: Tints, shades, tones
- [ ] **Semantic Colors**: Success, error, warning, info
- [ ] **Usage Rules**: When/where to use each color
- [ ] **Accessibility**: WCAG-compliant color combinations documented

### Typography System
- [ ] **Font Files**: OTF/TTF files + web font formats (WOFF2)
- [ ] **Font Licensing**: Documented (client can use legally)
- [ ] **Type Hierarchy**:
  - [ ] Display/headline (size, weight, usage)
  - [ ] Subheadings (H1-H6 scale)
  - [ ] Body text (size, line height, max width)
  - [ ] Captions/small text
- [ ] **Type Pairing Rules**: Which fonts work together
- [ ] **Fallback Fonts**: Web-safe alternatives

### Applications (Optional, depends on scope)
- [ ] Business cards (print-ready PDF)
- [ ] Letterhead template
- [ ] Email signature (HTML)
- [ ] Social media templates (profile, cover, post templates)
- [ ] Presentation template (PowerPoint/Keynote/Google Slides)
- [ ] Website mockups

---

## Print Design (Brochure, Poster, Flyer)

### Design Files
- [ ] **Source Files**: INDD, AI, or PSD with layers intact
- [ ] **Linked Assets**: All images, fonts included (package for INDD)

### Print-Ready Files
- [ ] **PDF/X-1a** or **PDF/X-4** format
- [ ] **Bleed**: 3mm (0.125") on all sides
- [ ] **Trim Marks**: Crop marks included (if requested)
- [ ] **Color Mode**: CMYK (or spot colors documented)
- [ ] **Resolution**: 300 DPI minimum
- [ ] **Fonts**: Embedded or outlined
- [ ] **Image Links**: No missing links
- [ ] **Safe Area**: Critical content 5mm from trim

### Specifications Document
- [ ] Final trim size (e.g., 8.5×11")
- [ ] Paper stock recommendation (weight, finish)
- [ ] Color process (CMYK, PMS, 4-color)
- [ ] Quantity recommendations
- [ ] Finishing options (lamination, UV coating, etc.)

### Proofs
- [ ] Soft proof (PDF for client review)
- [ ] Hard proof (if requested, from printer)

## Motion Graphics/Animation

### Project Files
- [ ] **Source Files**: AEP (After Effects), Blender, Cavalry
- [ ] **Assets**: All images, fonts, audio files

### Style Frames
- [ ] Key visual moments (3-5 frames showing look/feel)
- [ ] Exported as PNG/JPG (high-res)

### Storyboard
- [ ] Sequence of scenes (with annotations)
- [ ] Timing notes (duration per scene)

### Timing Guide
- [ ] Total duration
- [ ] Scene breakdown (timecode for each scene)
- [ ] Easing curves specified (ease-in, ease-out, etc.)

### Final Renders
- [ ] **Video Files**: MP4 (H.264), MOV (ProRes if high-quality needed)
- [ ] **Resolutions**:
  - [ ] 1920×1080 (Full HD, standard)
  - [ ] 3840×2160 (4K, if requested)
  - [ ] Social media sizes (1080×1920 for Stories, 1080×1080 for Instagram)
- [ ] **Frame Rate**: 24fps (cinematic), 30fps (standard), 60fps (smooth)
- [ ] **Bitrate**: High (10-20 Mbps for quality)

### Deliverables
- [ ] Final video (compressed for delivery: Vimeo, YouTube, social)
- [ ] Thumbnail image (for video preview)
- [ ] Subtitle/caption file (SRT if applicable)
- [ ] Audio file separate (if music/VO needs to be swapped)

## General File Organization

Deliver files in clear folder structure:

```
project-name/
├── 01-source-files/
│   ├── logo.ai
│   ├── brand-guidelines.indd
│   └── assets/ (fonts, images)
├── 02-final-exports/
│   ├── logo-primary.svg
│   ├── logo-primary.png
│   ├── brand-guidelines.pdf
│   └── print-ready.pdf
├── 03-specifications/
│   ├── specs.pdf
│   └── usage-guidelines.pdf
└── README.txt (instructions, file inventory)
```

## Handoff Checklist (Before Delivery)

- [ ] All files tested (open in appropriate software, no errors)
- [ ] File names clear and semantic (no "untitled-1.ai")
- [ ] README or instructions included
- [ ] Client can open all files (check software version compatibility)
- [ ] Fonts included or outlined (client doesn't need to purchase)
- [ ] Images optimized (web files compressed, print files high-res)
- [ ] Color modes correct (CMYK for print, RGB for screen)
- [ ] Accessibility checked (contrast, readability)
- [ ] Reviewed for errors (typos, broken links, missing assets)
- [ ] Organized in logical folder structure
- [ ] Compressed for delivery (ZIP file, or cloud link if large)

---

**Pro Tip:** Over-deliver slightly. Include a bonus item (extra social template, icon set, or quick reference card) to exceed expectations.
